Mood Based Musics Movies

Python CTkinter SQLite API

Aperçu du Projet

Mood Based Musics Movies (MBMM) est une application de recommandation qui suggère des films et des musiques en fonction de l'humeur de l'utilisateur. En combinant l'analyse des émotions et les préférences personnelles, MBMM crée des playlists et des listes de films parfaitement adaptées à l'état émotionnel du moment.

Contrairement aux systèmes de recommandation traditionnels qui se basent uniquement sur les goûts ou les historiques, MBMM intègre la dimension émotionnelle pour des suggestions plus pertinentes et contextuelles, enrichissant ainsi l'expérience utilisateur.

Fonctionnalités Principales

  • Détection de l'humeur via questionnaire interactif ou analyse faciale
  • Recommandations personnalisées de musiques via Spotify API
  • Suggestions de films via TMDB API en fonction de l'humeur
  • Création et gestion de playlists thématiques
  • Profils personnalisés avec historique des humeurs
  • Mode découverte pour explorer de nouveaux contenus

Détails d'Implémentation

L'application est structurée autour d'une architecture modulaire avec des composants distincts pour la détection d'humeur, les recommandations et l'interface utilisateur. Le backend utilise SQLite pour stocker les données utilisateur et les préférences, tandis que les API externes (Spotify, TMDB) sont intégrées via des wrappers Python optimisés.

Le module de détection d'humeur combine un système basé sur des questionnaires et une analyse d'expression faciale optionnelle (via OpenCV et un modèle de deep learning). Les algorithmes de recommandation utilisent un système hybride combinant filtrage collaboratif et analyse de contenu pour des suggestions précises.

L'interface utilisateur développée avec CTkinter (CustomTkinter) offre une expérience moderne et réactive, avec des transitions fluides et un design adaptatif.

Défis et Solutions

Cartographie des émotions aux contenus

Établir des correspondances précises entre les états émotionnels et les caractéristiques des films/musiques était un défi majeur. J'ai développé un modèle de classification multi-dimensionnel qui intègre plusieurs attributs (rythme, tonalité, genre, thèmes) et utilise des techniques d'apprentissage par renforcement pour s'améliorer avec les retours utilisateurs.

Limitations des API et gestion du cache

Les limites de taux des API externes et les temps de réponse variables posaient des problèmes de performance. J'ai implémenté un système de mise en cache intelligent qui précharge les recommandations probables en fonction des habitudes de l'utilisateur et stocke localement les données fréquemment utilisées, réduisant les appels API de 75% et améliorant significativement la réactivité de l'application.
